Core.async compared to Goroutines

By Markus Hjort / @mhjort

Tomorrow Tech

Tools for asynchronous programming based on the ideas of CSP
Designed to run on one physical machine
Simple high level API
Efficient memory usage (avoid system threads)

Goroutines example


c := make(chan string)
go func() { c <- "ping" }()
msg := <-c
fmt.Println(msg)

Goroutines design

Own scheduler for goroutines with dynamic (initially small) stack size

core.async example


(let [c (chan)]
  (go (<! c "ping"))
  (>!! c)
  (println c))

core.async design

Fixed size JVM thread pool and magical macro for transforming your code into a statemachine
